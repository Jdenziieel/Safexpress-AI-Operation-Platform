🧭 MODEL STRATEGY (last changed: 2026-04-24)
--------------------------------------------------------------------------------
Per the supervisor-agent model-change guide (the source of truth for token
accounting across this capstone):

  • Heavy, cacheable calls → gpt-4.1
      - ws_chat_stream (big KB system prompt repeats every turn)
      - chat_message (same)
      - pdf_parse chunking + image analysis (system prompt repeats per image)
    Why: 75% cache discount on the repeated system prompt + 1M context.

  • Short, never-repeated calls → gpt-4o-mini
      - session title generation
      - pronoun/follow-up resolver (QUERY_RESOLVER_MODEL)
      - chunk reranking (RERANK_MODEL)
    Why: prompt changes every call, so caching can't help. gpt-4o-mini has
    ~4x lower cold-prompt cost than gpt-4.1-mini on these shapes.

All of the above are env-overridable — flip the env var, no redeploy needed.

⚡ QUICK AWS CLI BATCH UPDATE (Replace YOUR_VALUES):

#!/bin/bash

FUNCTIONS=(
  "kb-list"
  "kb-delete"
  "kb-query"
  "kb-versions"
  "chat-session-create"
  "chat-message"
  "chat-sessions"
  "chat-session-update"
  "chat-quota"
  "admin-health"
  "admin-stats"
  "admin-documents"
)

ENV_VARS='{
  "DOCUMENTS_TABLE": "KB_Documents",
  "VERSIONS_TABLE": "KB_DocumentVersions",
  "SESSIONS_TABLE": "KB_ChatSessions",
  "MESSAGES_TABLE": "KB_ChatMessages",
  "LOGS_TABLE": "KB_Logs",
  "WEAVIATE_URL": "YOUR_WEAVIATE_URL",
  "WEAVIATE_API_KEY": "YOUR_WEAVIATE_KEY",
  "OPENAI_API_KEY": "YOUR_OPENAI_KEY",
  "OPENAI_MODEL": "gpt-4.1",
  "OPENAI_CHAT_MODEL": "gpt-4.1",
  "TITLE_MODEL": "gpt-4o-mini",
  "QUERY_RESOLVER_MODEL": "gpt-4o-mini",
  "RERANK_MODEL": "gpt-4o-mini",
  "EMBEDDING_MODEL": "text-embedding-3-small",
  "QUOTA_ENABLED": "false",
  "KB_FILES_BUCKET": "capstone-kb-files"
}'

for FUNC in "${FUNCTIONS[@]}"; do
  echo "Updating $FUNC..."
  aws lambda update-function-configuration \
    --function-name "$FUNC" \
    --environment "Variables=$ENV_VARS"
done

echo "✅ All 12 functions updated!"
